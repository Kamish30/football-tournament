# ⚽ Савося — Система управления футбольным турниром

Веб-приложение для учёта результатов любительского футбольного турнира. Работает в браузере на телефоне и компьютере, несколько операторов вносят результаты одновременно, зрители видят статистику в реальном времени.

**Сайт:** https://savosya.up.railway.app/

## Возможности

**Управление данными**
- Группы по годам рождения (уникальные — защита от дублей), URL вида `/group/2014`
- Команды с логотипами (до 100КБ), игроки с номерами
- Три роли: администратор, редактор, тренер + зрители без входа
- Тренеры управляют составами только назначенных команд
- Автогенерация расписания (все пары круга одной кнопкой)
- Два способа ввода результатов: карточка матча или быстрый ввод (+/− голы/ассисты)
- Оптимистичная блокировка — защита от конфликтов при одновременном редактировании

**Статистика**
- Турнирная таблица с формой последних 5 матчей (цветные кружки)
- Шахматка — перекрёстная таблица результатов с цветами
- Отдельный рейтинг баттлов вратарей
- Рейтинги бомбардиров и ассистентов (кликабельные карточки)
- Карточки команд: статистика, серия, лучшие игроки, таблицы по датам, состав, история
- Карточки игроков: голы, ассисты, история матчей
- Поиск игроков по имени

**Интерфейс**
- Дашборд на главной: последние результаты + ближайшие матчи
- Фильтры расписания по статусу и команде
- Хроника матча (таймлайн событий по командам)
- Тёмная тема с переключателем
- Логотип турнира в шапке и баннере
- Favicon, Open Graph мета-теги для мессенджеров

**Экспорт**
- CSV (Excel): таблица, шахматка, матчи, бомбардиры, ассистенты, игроки
- CSV по командам: бомбардиры/ассистенты по датам
- PNG-картинки для соцсетей с брендированием
- Бэкап базы данных одной кнопкой

## Быстрый старт (локально)

```bash
# 1. Установить Python 3.10+
# 2. Установить зависимости
pip install -r requirements.txt

# 3. Запустить
uvicorn main:app --reload --port 8000

# 4. Открыть http://localhost:8000
# Логин по умолчанию: admin / admin123
```

## Деплой на Railway

### Шаг 1: Загрузить проект на GitHub
```bash
cd tournament
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШЛОГИН/football-tournament.git
git push -u origin main
```

### Шаг 2: Развернуть на Railway
1. railway.app → войти через GitHub
2. **New Project** → **Deploy from GitHub repo**
3. Railway автоматически обнаружит Procfile

### Шаг 3: Настроить Volume (обязательно — иначе данные пропадут при деплое)
```bash
railway volume add --mount-path /data
railway variables set DATA_DIR=/data
```

### Шаг 4: Настроить секретный ключ
```bash
railway variables set SECRET_KEY=ваша-длинная-случайная-строка
```

### Шаг 5: Деплой
```bash
railway up
```

После каждого обновления кода: `railway up`, затем Ctrl+Shift+R в браузере (сброс кеша). Данные в Volume не затрагиваются при деплое.

## Переход на российский сервер (Timeweb Cloud)

Railway не принимает оплату из России. Рекомендуется Timeweb Cloud.

**Домен:** timeweb.com/ru/services/domains/ — от 179 руб/год (.ru)

**Вариант А — Cloud Apps (проще, от 199 руб/мес):**
1. Apps → Backend → FastAPI → подключить GitHub
2. Команда запуска: `uvicorn main:app --host 0.0.0.0 --port 80`
3. Переменные: `SECRET_KEY`, `DATA_DIR=/data`
4. Подключить Volume `/data`

**Вариант Б — VPS (от 300 руб/мес, больше контроля):**
1. Ubuntu 22.04, `pip install -r requirements.txt`
2. Systemd-сервис с `Environment=DATA_DIR=/opt/tournament/data`
3. Nginx + certbot для HTTPS

**Перенос данных:** скачайте бэкап с сайта (кнопка на главной для admin) → переименуйте в `tournament.db` → положите в папку данных нового сервера.

## Учётные записи

| Роль | Что может |
|------|-----------|
| **admin** | Всё + пользователи, назначение команд тренерам, бэкап БД |
| **editor** | Группы, команды, логотипы, матчи, результаты, генерация круга, статистика |
| **coach** | Управление составами только назначенных команд (любые группы) |
| **зритель** (без входа) | Просмотр всего, экспорт, поиск, карточки |

Логин по умолчанию: `admin` / `admin123` — **смените пароль при первом деплое!**

## Структура проекта

```
tournament/
├── main.py              # FastAPI приложение, все API-эндпоинты
├── db.py                # Модели БД (SQLAlchemy + SQLite), миграции
├── auth.py               # Авторизация (JWT, хеширование паролей)
├── schemas.py            # Валидация данных (Pydantic)
├── requirements.txt      # Python зависимости
├── Procfile               # Команда запуска для Railway
├── static/
│   ├── style.css          # Стили (голубая тема, тёмная тема, адаптив)
│   ├── app.js             # API-клиент, авторизация, тема
│   └── logo.png            # Логотип турнира
└── templates/
    ├── index.html         # Главная — группы, дашборд, поиск, тренер
    ├── login.html          # Страница входа
    └── group.html          # Страница группы — все вкладки, экспорт
```

## Технологии

- **Backend:** Python 3.13, FastAPI, SQLAlchemy, SQLite (WAL-режим)
- **Frontend:** Vanilla JS, CSS (Inter, голубая тема из логотипа)
- **Авторизация:** JWT токены (PBKDF2 + HMAC-SHA256, 72 часа)
- **Хостинг:** Railway (Volume для персистентности данных)
